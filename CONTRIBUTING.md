# Contribution Guide

## English

Thank you for considering contributing to this repository.

Before creating an Issue or Pull Request, please contact us on Discord first. To help things move smoothly, we ask contributors to discuss their plans on Discord in advance in principle.
Fraxinus 00tw is not operated as a fully open development project on GitHub alone.
The design data published in this repository is open, but the development workflow itself is not open for unrestricted drive-by contribution.

Discord:
https://fraxinus.jp/discord

### Exceptions: Pull Requests you may open without prior Discord discussion

As an exception, you may open a Pull Request without prior Discord discussion for minor fixes such as the following.
However, even in these cases, merge is not guaranteed. Each Pull Request will be reviewed for consistency with the project direction and maintenance policy.

- Documentation typo fixes
- Obvious mistakes in file names or headings
- Broken links or inconsistent wording
- Wording cleanup that does not change meaning

### Changes that should be discussed on Discord first

Please discuss the following on Discord before creating an Issue or Pull Request.

- Specification changes
- Design changes
- Adding new parts
- Changing the shape of existing parts
- Structural or directory organization changes
- Changes that affect assembly instructions or project operation
- Bug reports or improvement proposals

### If you open a Pull Request after prior agreement

If you open a Pull Request after prior agreement on Discord or elsewhere, you are understood to agree that the submitted contents will be published under the following terms.

- Documentation and images will be published under CC BY-NC-SA
- Design data, scripts, and other design-related data will be published under GPLv3

Credits may be listed in the README or other documentation.
However, for small changes such as typo fixes or minor documentation edits, we appreciate the contribution but may choose not to list a formal credit.
Whether and how credits are listed is decided by the core maintainers.

### Issues

Before opening an Issue, please share the matter on Discord first in principle.
Create an Issue when discussion on Discord leads to the conclusion that the topic should be tracked on GitHub.
Issues or Pull Requests created without prior discussion, or submissions that do not follow this guide, may be closed after guidance is provided.

### Editing README files

README files in this repository are maintained in Japanese.
If you edit `README.md`, you must also update the corresponding `README.en.md` with the same content.

## 日本語

このリポジトリへのコントリビューションを検討していただき、ありがとうございます。

Issue や Pull Request を作成する前に、まずは Discord でご相談ください。よりスムーズに進めるため、原則として Discord での事前相談をお願いしています。
Fraxinus 00tw では、GitHub 上のやり取りだけで完結するコントリビューションは基本的に想定していません。
このリポジトリで公開している設計データはオープンですが、開発フローそのものをオープンに受け付ける運用ではありません。

Discord:
https://fraxinus.jp/discord

### 例外として直接 Pull Request を出せるもの

以下のような軽微な修正は例外として、Discord での事前相談なしに Pull Request を作成して構いません。
ただし、これらの Pull Request についても、内容や方針との整合性を確認したうえで判断するため、マージを保証するものではありません。

- ドキュメントの typo 修正
- ファイル名や見出しなどの明らかな誤記修正
- リンク切れや表記ゆれの修正
- 説明文の意味を変えない範囲での文面整理

### 事前に Discord で相談してほしいもの

以下は、Issue や Pull Request を作成する前に Discord で相談してください。

- 仕様変更
- 設計変更
- 新規パーツの追加
- 既存パーツの形状変更
- 構成やディレクトリ整理に関わる変更
- 組み立て手順や運用に影響する変更
- 不具合報告や改善提案

### 合意のうえで Pull Request を作成する場合

Discord などで事前に合意したうえで Pull Request を作成する場合、その Pull Request に含まれる内容は以下の条件で公開されることに同意したものとみなします。

- ドキュメントと画像は CC BY-NC-SA で公開されます
- 設計データ、スクリプト、その他の設計関連データは GPLv3 で公開されます

クレジットは README や各種ドキュメント内に記載する場合があります。
ただし、typo 修正や軽微な文書修正などの小さな変更については、感謝はしますが、クレジットとして明記するとは限りません。
クレジットの記載有無や記載方法は、コアメンテナが判断します。

### Issue について

Issue を立てる前にも、原則として Discord で状況共有をお願いします。
Discord で整理した結果、GitHub 上で追跡するのが適切と判断された場合に、Issue を作成してください。
事前相談なしの Issue や Pull Request、またはこのガイドに沿っていない内容については、案内のうえで close する場合があります。

### README の編集について

このリポジトリの README は日本語で管理しています。
`README.md` を編集する場合は、対応する `README.en.md` も同じ内容に更新してください。
